1. Enter the repository

Give us the repository name as well as the organisation or user under which it lives. For example, pandas can be found with "pandas-dev/pandas".

2. Grab a coffee

We'll clone the repository to crunch some vital statistics, and then combine these with data from GitHub's API.

3. Analyse

We don't provide a health "score", instead we present a number indicators of repository health and leave you to draw your own conclusions.

4. Extend

Your analysis doesn't need to stop there. You can use our API to fetch the data to do your own analysis. If you'd like to contribute a health indicator to the repo-health-report pull-requests are welcome at pelson/repo-health-report
